跨域（非同源策略请求）

广义的跨域

  • 资源跳转：a链接、重定向、表单提交
  • 资源嵌入：<link> <script> <img> <iframe>等DOM标签，还有样式中的background:url() @font-face()等文件外链不存在跨域请求的限制
  • 脚本请求：js发起的ajax请求、dom和js对象的跨域操作

同源策略（狭义的）

  • 同源策略：是浏览器的一种安全策略，同源指的是请求URL地址中的协议、域名和端口都相同，其中之一不相同就是跨域
  • 同源策略主要是为了保证浏览器的安全性
  • 同源策略下，浏览器不允许Ajax跨域获取服务器数据
  • 同源策略限制以下几种行为
    • Cookie、LocalStorage 和 IndexDB 无法读取
    • DOM和JS对象无法获得
    • AJAX 请求不能发送

跨域解决方案

通过jsonp跨域

  • 原生js实现
    <script>
  // 动态创建script标签 也可以在script标签的src属性中直接指定请求地址 不太灵活
  // 动态创建script标签是异步请求
  var script = document.createElement('script');
  script.type = 'text/javascript';
  var username = 'admin'
  // 指定后端返回并执行的回调函数的名称
  script.src = 'http://www.domain.com/login?username='+username+'&callback=handerCallback';
  document.head.appendChild(script);
  // 回调函数 由服务端响应的内容调用
  function handlerCallback(data) {
    console.log(data);
  }
   // 服务端的返回的内容是一个函数调用
  handlerCallback({"username":"admin","status":200})
</script>
  • jQuery实现
    $.ajax({
  type: 'get',
  url: 'http://www.domain.com/login',
  dataType: 'jsonp', // 必须指定jsonp格式 只支持get请求
  data: {},
  jsonp: 'cb', // 默认值为callback '='前面的值
  jsonpCallback: 'handlerCallback', // 自定义回调函数的名字 '='后面的值
  success: function(data) {
  },
  error: function(err) {
  }
})

postMessage

  • 可以解决以下问题

    • 多窗口之间消息传递
    • 页面和其打开的新窗口的数据传递
    • 页面与嵌套的iframe消息传递
    • 上面三个场景的跨域数据传递

  • 页面一（www.test1.com/a.html）

    <iframe id="iframe" src="http://www.test2.com/b.html" style="display:none;"></iframe>
<script>       
    var iframe = document.getElementById('iframe');
    iframe.onload = function() {
        var data = {
            name: 'Kobe'
        };
        // 发送跨域数据
        iframe.contentWindow.postMessage(JSON.stringify(data), 'http://www.test2.com');
    };

    // 接受返回的数据
    window.onmessage = function(e) {
        // e.data的是接收到的信息
        console.log('data from test2 ---> ' + e.data);
    }
</script>

  • 页面二（www.test1.com/b.html）

    <script>
// 接收test1的数据
window.onmessage = function(e) {
    alert('data from test1 ---> ' + e.data);
    var data = JSON.parse(e.data);
    if (data) {
        data.age = 40;
        // 处理后再发回test1
        window.parent.postMessage(JSON.stringify(data), 'http://www.test1.com');
    }
}
</script>

WebSocket协议跨域（客户端和服务端实时通信）

  <div>user input：<input type="text"></div>
  <script src="https://cdn.bootcss.com/socket.io/2.2.0/socket.io.js"></script>
  <script>
    var socket = io('http://www.test2.com:8080');
    // 连接成功处理
    socket.on('connect', function() {
        // 监听服务端消息
        socket.on('message', function(msg) {
            console.log('data from server: ---> ' + msg); 
        });
        // 监听服务端关闭
        socket.on('disconnect', function() { 
            console.log('Server socket has closed.'); 
        });
    });
    document.getElementsByTagName('input')[0].onblur = function() {
        socket.send(this.value);
    };
  </script>

document.domain + iframe

  • 应用场景：只能实现同一个主域，不同子域的跨域应用场景
  • 实现原理：两个页面都通过js强制设置document.domain为基础主域，就实现了同域
  • 父页面（www.test.com/a.html）
    <iframe id="iframe" src="http://school.test.com/b.html"></iframe>
<script>
    document.domain = 'test.com';
    var data = 'Kobe';
</script>
  • 子页面（school.test.com/b.html）
    <script>
  document.domain = 'test.com';
  // window.parent ==> b页面的父页面就是a
  // 获取父窗口中变量 
  console.log('get js data from parent ---> ' + window.parent.data);
</script>

window.name + iframe

  • 页面一（www.test1.com/a.html）
    <iframe id="iframe" src="http://www.test2.com/b.html" style="display:none;"></iframe>
<script>       
    var iframe = document.getElementById('iframe');
    var count = 0;
    iframe.onload = function() {
      // 需要把地址重新指向到同源的页面二才可以 这样onload执行两次
      if (count === 0) {
        iframe.src = 'http://www.test1.com/proxy.html';
        count++;
        return;
      }
      console.log(iframe.contentWindow.name); // Kobe
    };
</script>
  • 页面二（www.test1.com/proxy.html）：中间代理页，与页面一同域，页面不用写东西
  • 页面三（www.test2.com/b.html）
    <script>
  window.name = 'Kobe';
</script>

location.hash + iframe

  • 页面一（www.test1.com/a.html）

    <iframe id="iframe" src="http://www.test2.com/b.html" style="display:none;"></iframe>
<script>
    var iframe = document.getElementById('iframe');
    setTimeout(function() {
      // 向b.html传hash值
      // iframe.src = iframe.src + '#name=kobe';
      iframe.src = 'http://www.test2.com/b.html#name=kobe';
    }, 1000);

    // 开放给同域c.html的回调方法
    function callback(res) {
      alert('data from c.html ---> ' + res);
    }
</script>

  • 页面二（www.test2.com/b.html）

    <iframe id="iframe" src="http://www.test1.com/c.html" style="display:none;"></iframe>
<script>
    var iframe = document.getElementById('iframe');
    // 监听a.html传来的hash值，再传给c.html
    window.onhashchange = function () {
        iframe.src = iframe.src + location.hash;
    };
</script>

  • 页面三（www.test1.com/c.html）和页面一同源

    <script>
  // 监听b.html传来的hash值
  window.onhashchange = function () {
      // 再通过操作同域a.html的js回调，将结果传回c
      window.parent.parent.callback('hello: ' + location.hash.replace('#name=', ''));
  };
</script>
