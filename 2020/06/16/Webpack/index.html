Webpack（打包之后就没有用了）的安装

  • 先安装node

  • 查看node版本
    node -v

  • 全局安装Webpack
    npm install webpack@版本号 -g

  • 本地安装Webpack 和项目同步
    npm install webpack@版本号 --save-dev（–save-dev是开发时依赖，项目打包后不需要继续使用）

  • 查看webpack版本
    webpack --version

    Webpack文件夹解析

  • dist文件夹：用于存放之后打包的文件

  • src文件夹：用于存放源文件

  • index.html：首页

  • package.json：通过npm init生成的，npm包管理的文件

  • webpack.config.js：配置文件（文件名不可随意修改） 

      // npm init   npm install
  // 导入node中的path模块
  const path = require('path');
  // 映射webpack '入口文件' '出口文件'
  module.exports = {
      entry: './src/index.js',
      output: {
          path: path.join(__dirname,'dist'),
          filename: 'bundle.js'
      }
  }

  • package.json：依赖文件 

      {
      "name": "meetwebpack",
      "version": "1.0.0",
      "description": "",
      "main": "webpack.config.js",
      "scripts": {
          "test": "echo \"Error: no test specified\" && exit 1",
          // 用于 npm run build映射 相当于运行webpack命令 会优先去本地找webpack，然后再去全局找，只要在终端中运行webpack就是全局
          "build": "webpack"
      },
      "author": "",
      "license": "ISC"
      // 开发时依赖（开发阶段使用）
      "devDependencies": {
          "webpack": "^3.6.0"
      }
      // 打包后使用的依赖
      "dependencies": {

      }
  }
