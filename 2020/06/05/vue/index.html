-Vue基础

Vue安装的三种方式

  1. CDN引入

    • 开发环境版本，包含了有帮助的命令行警告
      <script src="https://cdn.jsdelivr.net/npm/vue/dist/vue.js"></script>
    • 生产环境版本，优化了尺寸和速度
      <script src="https://cdn.jsdelivr.net/npm/vue"></script>

  2. 官方下载引入

    • 直接引入<script>标签

  3. NPM安装

    • npm install vue

创建Vue实例传入的options

  • el
    • 类型：string | HTMLElement
    • 作用：决定之后Vue实例会管理哪一个DOM
  • data
    • 类型：Object | Function
    • 作用：Vue实例对应的数据对象
  • methods
    • 类型：{ [key: string]: Function }
    • 作用：定义属于Vue的一些方法，可以在其他地方调用，也可以在指令中使用

插值操作

  1. Mustache语法 
    <div id="app">
   <h2>{{hello}}</h2>
   <h2>{{hello + ' ' + world}}</h2>
   <h2>{{hello}} {{world}}</h2>
   <h2>{{count * 2}}</h2>
</div>
<script>
   const app = new Vue({
      el: '#app',
      data: {
         hello: 'hello',
         world: 'world',
         count: 100
      }
   })
</script>
  2. v-once指令
    • 该指令后不需要跟任何表达式
    • 该指令表示元素和组件只渲染一次，不会随着数据的该笔而改变
  3. v-html指令
    • 该指令后面往往跟上一个string类型
    • 会将string类型的html解析并渲染
  4. v-text指令
    • 该指令与Mustache语法比较相似，都是将数据显示在界面，但不够灵活
    • 该指令接受一个string类型
  5. v-pre指令
    • 该指令后不需要跟任何表达式
    • 该指令跳过解析，显示原本的内容
  6. v-cloak指令
    • 该指令后不需要跟任何表达式
    • 斗篷

v-bind指令

  • 该指令可以动态绑定属性
    1. 使用方法 
      <div id="app">
<img v-bind:src="imgURL" alt="">
</div>
    2. 语法糖 
      <div id="app">
<img :src="imgURL" alt="">
</div>
  • 动态绑定class
    1. 使用方法 
      <style>
.active {
  color: red;
}
</style>
<div id="app">
<h2 :class="active">hello</h2>
<h2 :class="{类名1:boolean,类名2:boolean}">hello</h2>
<h2 :class="{key(css属性名):value(css属性值)}">hello</h2>
</div>

计算属性

  1. 使用方法 

    <div id="app">
  <h2>{{fullStr}}</h2>
</div>

const app = new Vue({
  el: '#app',
  data: {
    firstStr: 'hello',
    lastStr: 'world'
  },
  computed: {
    fullStr() {
     return this.firstStr + ' ' + this.lastStr;
    }
  }
})

  2. setter和getter方法 

    <div id="app">
  <h2>{{fullStr}}</h2>
</div>

const app = new Vue({
  el: '#app',
  data: {
    firstStr: 'hello',
    lastStr: 'world'
  },
  computed: {
    fullStr: function() {
      set: function(value) { //大多数情况不写
      const str = value.split(' ');
      this.firstStr = str[0];
      this.lastStr = str[1];
      },
      get: function() {
        return this.firstStr + ' ' + this.lastStr;
      }
    }
  }
})

  3. computed和methods的对比

    • 多次使用时computed比methods性能高
    • Vue内部对computed做了缓存，多次使用只调用一次，只有内容发生改变时，才会重新调用、缓存；而methods是使用几次调用几次

v-on事件监听

  • 绑定事件监听器
  • 使用 
    <button v-on:click=""></button>
  • 语法糖 
    <button @click=""></button>
  • v-on参数
    1. 若方法不需要额外参数，方法后的()可以不添加
    2. 写方法时省略了()，但方法本身是需要一个参数的Vue会默认将浏览器生产的event对象作为参数传入
    3. 方法定义时，需要event对象，也需要其他参数
      <button @click="btnClick(param,$event)"></button>
  • v-on修饰符
    1. 阻止冒泡stop 
       <div @click="divClick">
   <button @click.stop="btnClick">按钮</button>
 </div>
    2. 阻止默认行为prevent 
       <form action="baidu">
   <input type="submit" value="提交" @click.prevent="submitClick">
 </form>
    3. 监听某个键盘键帽的点击(例如enter键)
       <input type="text" @keyup.enter="keyClick">
    4. 只触发一次回调once
       <button @click.once="buttonClick">按钮</button>

条件判断

  • v-if的使用(接收一个Boolean值)
    <h2 v-if="true">hello world</h2>
  • v-else的使用(v-if接收的是false是起作用)
    <h2 v-if="isShow">isShow为true时显示</h2>  
<h2 v-else>isShow为false时显示</h2>
  • v-else-if的使用(不建议)
    <p v-if="score>=90">优秀</p>
<p v-else-if="score>=80">良好</p>
<p v-else-if="score>=60">合格</p>
<p v-else>不及格</p>
  • 这里补充一下Vue中的Virtual DOM，也就是所谓的虚拟DOM
    • Virtual DOM其实就是一棵以JavaScript对象(VNode 节点)作为基础的树，用对象属性来描述节点，实际上它只是一层对真实 DOM 的抽象。最终可以通过一系列操作使这棵树映射到真实环境上。简单来说，可以把Virtual DOM 理解为一个简单的JS对象，并且最少包含标签名(tag)、属性(attrs)和子元素对象(children)三个属性。不同的框架对这三个属性的命名会有点差别
    • Virtual DOM在Vue中做了什么
      1. 供与真实DOM节点所对应的虚拟节点Vnode
      2. 将虚拟节点Vnode和旧虚拟节点oldVnode进行对比，然后更新视图
  • v-show的使用
    <h2 v-if="isShow">isShow为true时显示</h2>
<h2 v-show="isShow">isShow为true时显示</h2>
    • v-if与v-show控制元素显示的区别
      1. v-if，条件为false时，元素不会在DOM中存在；条件为true时，会重新创建元素
      2. v-show，条件为false时，只是增加了行内样式：display:none;元素仍在DOM中存在
      3. 切换评率较高时使用v-show，只有一次使用v-if

循环遍历

  • v-for遍历数组
    <ul>
  <li v-for="(item,index) in books">{{index+1}}-{{item}}</li>
</ul>
  • v-for遍历对象
    <ul>
  <li v-for="(value,key,index) in info">{{key}}-{{value}}-{{index}}</li>
</ul>
  • 数组中哪些方法是响应式的？
    1. push():在数组末尾添加元素
    2. pop():删除数组最后一个元素
    3. shift():删除数组第一个元素
    4. unshift():在数组最前面添加元素
    5. splice():可删除/替换/插入元素
    6. sort():排序
    7. reverse():反转
    8. Vue.set(要修改的对象,修改的索引值,修改后的值)
    9. 通过索引值修改数组中的元素不可以做到响应式
