-Vue-Router -前端路由

什么是前端渲染？什么是后端渲染？

什么是前后端分离？

  • 有了Ajax的出现，才有了前后端分离的开发模式

  • 后端只提供数据API，前端通过Ajax获取数据，并通过JavaScript将数据渲染到页面

  • 优势：前后端责任清晰，前端专注于交互和可视化，后端专注于数据

  • 移动端出现，后端的API不需要任何处理

    什么是前端路由？什么是后端路由

    单页面富应用（SPA）

  • 在前后端分离的基础上加了一层前端路由

  • 由前端来维护一套路由规则

    如何在改变URL的同时让页面不刷新

    • 通过location.hash = 'foo'
      http://localhost:8080/#/foo
    • 通过h5中的history.pushState({},'','home')修改（类似栈结构）
      http://localhost:8080/home
    • 通过h5中的history.replaceState({},'','home')修改（替换）
    • 返回history.back()等价于history.go(-1)；向前history.forward()等价于history.go(1)

      三大框架的路由实现

    • Angular中的ngRouter
    • React中的ReactRouter
    • Vue中的vue-router

      vue-router基于路由和组件

    • 路由用于设定访问路径，将路径和组件映射起来
    • 在SPA中，页面路径的改变就是组件的切换

      安装和使用vue-router

    • 通过npm install vue-router --save安装
    • 在模块化工程中使用它

  • 导入路由对象，并调用Vue.use(Router)

  • 创建路由实例，传入路由映射配置

  • 在Vue实例中挂载创建的路由实例

    • 使用vue-router的步骤

  • 创建路由组件并配置路由映射

    export default new Router({
  routes: [
    {
      path: '/home',
      component: Home
    },
    {
      path: '/about',
      component: About
    }
  ]
})

  • 挂载路由

    new Vue({
  el: '#app',
  router,
  render: h => h(App)
})

  • 使用，通过<router-link>和<router-view>

    <div id="app">
  <router-link to="/home">to home</router-link>
  <router-link to="/about">to about</router-link>
  <router-view></router-view>
</div>
    • 路由的默认值和修改history模式（默认hash模式）
      routes: [
{
path: '',
redirect: '/about'
}
],
mode: 'history'
    • <router-link>的其他属性

  • tag：可以指定之后渲染成什么组件
    <router-link to="/home" tag="button"></router-link>

  • replace：replace不会留下历史记录，也就是指定replace后，返回键不能返回上一个页面

  • active-class：当<router-link>对应的路由匹配成功时，会自动给当前元素设置一个router-link-active的class，要想修改只需加入active-class=""或者再routes中加入linkActiveClass: ''属性

    路由的懒加载

    • 当打包构建应用时，JavaScript包会变得非常大，影响页面加载
    • 把不同路由对应的组件分割成不同的的代码块，当路由访问的时候才加载对应组件，会更高效
    • 路由懒加载的方式

  • 方式一：结合Vue的异步组件和Webpack的代码分析（看着就烦）

    const Home = resolve => {
  require.ensure(['../components/Home.vue'],()=> {
    resolve(require('../components/Home.vue'))
  })
}

  • 方式二：AMD写法

    const Home = resolve => require(['../components/Home.vue'],resolve)

  • 方式三：ES6中的简单写法

    const Home = () => import('../components/Home.vue')
    • 路由的嵌套
      {
 path: '/home',
 component: Home,
 children: [
   {
     path: '',
     redirect: 'news'
   },
   {
     path: 'news',
     component: News
   },
   {
     path: 'message',
     component: Message
   }
 ]
}
    • 路由传递参数

  • 通过param类型

    • 配置路由格式：/router/:id
    • 传递方式：在path后面跟上对应的值
    • 传递后形成的路径：/router/123
      {
path: '/user/:userId',
component: User
}
<router-link v-bind:to="'/user/'+userId" replace>用户</router-link>
// 获取userId
$route.params.userId

    ```

  • 通过query类型

    • 配置路由格式（普通配置）：/router/
    • 传递方式：对象中使用query的key作为传递方式
    • 传递后形成的路径：/router?id=123
      {
path: '/profile',
component: Profile
}
<router-link :to="{path: '/profile', query: {name: 'xxx',age: 18,gender: 'man'}}" replace>档案</router-link>
// 获取query
$route.query
